# 📚 本地文档检索系统（Doc Searcher）

基于 Rust 与 egui 开发的桌面全文检索引擎，支持 PDF / DOCX / TXT 文件索引、关键词搜索、页码定位，具备多数据库管理、实时文件监控、索引对比、导出 CSV 等功能。
在使用类似的软件 ANYTXT的时候觉得，完全实时全磁盘搜索导致电脑太卡了，所以完全用 DEEPSEEK 编程复刻了主要的几个功能，指定文件夹，按需要检索，增加词频排序，自用够用了，后面不定期再加几个功能完善，修复BUG。至于为什么用rust不用python, rust实在太快了。

---

## 🚀 功能特性

- **多数据库支持**：可创建多个独立索引库，分别管理不同目录的文档。
- **全文索引**：基于 Tantivy 的高性能搜索引擎，支持中文分词（jieba）。
- **页面感知**：自动识别 PDF / DOCX 的页码，跳转时直接将页码传递给外部阅读器。
- **实时监控**：索引完文件后，自动监听目录变化，更新索引。
- **排序与视图**：支持按相关度、文件名、修改时间、词频排序；紧凑/详细双视图。
- **导出结果**：将搜索结果导出为 CSV 文件。
- **索引对比**：对比磁盘文件与当前索引，显示新增、修改、删除的文件数量。
- **日志系统**：详细记录操作过程，方便排查问题。
- **可配置 PDF 阅读器**：自定义命令模板，支持 `{file}` `{page}` `{keyword}` 占位符。
- **快捷键支持**：`Ctrl+F` 聚焦搜索框，`Esc` 清除搜索。

## 🗂️ 配置与目录结构

所有数据存储于 `~/.doc_searcher/`：

```
~/.doc_searcher/
├── config.json          # 配置文件
├── error.log            # 错误日志
└── db/                  # 数据库目录
    └── <db_id>/         # 每个数据库的索引和元数据
        └── data/        # index/ & meta/
```

**配置文件 `config.json` 主要字段**：

- `base_dir`: 数据库根目录（默认 `.doc_searcher`）
- `pdf_reader`: PDF 阅读器命令行模板
- `ignore_case`: 是否忽略大小写（默认 true）
- `search_history`: 搜索历史记录
- `result_view`: 结果视图模式（`Compact` 或 `Detailed`）
- `font_scale`: 界面缩放比例
- `databases`: 所有数据库信息列表
- `active_db_id`: 当前激活的数据库 ID

## 🏁 快速入门

1. 启动程序。
2. 点击 **📂 管理数据库** → **新建数据库**，输入名称、简介，选择目标文件夹（包含 PDF/DOCX/TXT 的目录）。
3. 创建后自动激活该数据库，点击 **▶ 开始索引**，等待索引完成。
4. 在搜索框输入关键词，按回车或点击 **🔍搜索**。
5. 在结果列表中双击某项（或点击 📍）即可用关联程序打开文件。

## 🗄️ 数据库管理

点击工具栏 **📂 管理数据库** 打开管理窗口。

### 创建数据库
1. 输入 **名称** 和 **简介**。
2. 点击 **选择目标文件夹**，选定需要索引的根目录。
3. 点击 **创建并激活**。系统会自动在 `~/.doc_searcher/db/` 下生成索引目录，并立即激活。

### 激活数据库
在数据库列表中点击 **激活** 按钮，当前数据库会被切换，之前的索引保持不动。

### 删除数据库
点击 **删除** 按钮，将彻底移除该数据库的索引文件和配置。

---

## 🔍 索引

### 开始索引
激活数据库后，点击工具栏 **▶ 开始索引**。程序会遍历目标目录下所有支持文件：
- 计算文件 MD5 和修改时间，与已有索引对比。
- 对新增或变更的文件提取文本并分词，存入索引。
- 进度条实时显示进度。

### 暂停 / 恢复 / 停止
- **⏸ 暂停**：暂停索引，可稍后恢复。
- **▶ 恢复**：继续未完成的索引任务。
- **⏹ 停止**：终止索引，已索引的数据保留。

### 重建索引
在设置窗口中点击 **重建当前索引**，将清空当前数据库的所有索引并重新扫描，适用于切换了“忽略大小写”等需要重建的设置。

## 🔎 搜索

在激活的数据库中，搜索框输入关键词即可搜索。

### 基本搜索
- 输入任意中文或英文词汇，支持空格分隔多个词。
- 勾选 **忽略大小写** 可忽略英文大小写区别（默认开启）。
- 点击搜索框右侧的 **🔍搜索** 或按回车执行。

### 高级查询语法
程序底层使用 Tantivy，支持：
- 通配符：`*`（任意字符）、`?`（单个字符）
- 布尔操作：`AND`、`OR`、`NOT`
- 短语匹配：用双引号括起来，如 `"机器学习"`
- 点击 **💡** 可查看提示。

### 排序
工具栏提供排序方式：
- 🔥 相关度（默认）
- 📄 文件名 ↑ / ↓
- 🕒 最新 / 最早（按文件修改时间）
- 🔢 词频（关键词出现次数）

### 查看结果
- 左栏显示搜索结果，每项显示文件名、片段、命中次数。
- 点击某行选中，右侧 **命中详情** 列表会显示该文件中每个关键词的具体位置和上下文。
- 双击结果或点击 📍 按钮直接打开文件。
- 点击详情中的 📍 按钮可跳转到具体命中位置（页码/段落）。

### 导出结果
点击 **📤 导出 CSV**，选择保存路径，将当前搜索结果导出为 CSV 表格，包含文件名、路径、页码/段落、分数、命中次数、修改时间、文件大小、摘要。

## 📂 打开文件与定位页面

程序支持将搜索到的文档用外部程序打开，并**自动跳转到命中的页码/段落**。

### 默认打开方式
未配置 PDF 阅读器时，系统会使用操作系统的默认程序打开文件（不会传递页码）。

### 配置自定义 PDF 阅读器
在 **⚙️ 设置** 窗口的 **PDF阅读器命令** 中输入命令行模板，例如：

- **SumatraPDF**：  
  `SumatraPDF.exe -page {page} "{file}"`
- **Foxit Reader**：  
  `FoxitReader.exe /A page={page} "{file}"`
- **Adobe Acrobat**：  
  `Acrobat.exe /A page={page} "{file}"`
- **Edge 浏览器**：  
  `msedge.exe file:///{file}#page={page}`

其中：
- `{file}` 被替换为文件绝对路径，
- `{page}` 被替换为命中的页码（对 DOCX 为段落编号），
- `{keyword}` 为关键词（可用于部分阅读器的高亮）。

设置后，**打开 PDF 文件时将优先使用自定义命令**；其他类型文件仍用系统默认方式打开。

### 页码准确度说明
- **PDF**：程序会按页提取文本，索引中的页码应与原始 PDF 页面一一对应。如果发现页码跳转不正确，请检查 PDF 文本提取质量（可尝试更换 PDF 提取后端）。
- **DOCX**：段落编号即为 `location`，可作为跳转参考。
- **TXT**：页码固定为 1。

## 📁 实时文件监控

索引完成后，程序会在后台自动监控文档目录的变化：
- 文件被**修改**或**新增**时，自动更新索引。
- 监控在**索引开始时自动启动**（如果尚未启动），无需手动操作。

## 📊 索引对比

点击工具栏 **📊 对比** 按钮，程序会扫描当前目录并与数据库记录进行比较，显示：
- **新增**：磁盘存在但索引中不存在的文件数。
- **修改**：文件时间或 MD5 发生变化。
- **删除**：索引中存在但磁盘已不存在的文件数。

该功能可用于评估索引与文件系统的同步状态。

## ❓ 常见问题

### 1. 搜索无结果怎么办？
- 确保已完成索引（工具栏显示“已索引”数量 > 0）。
- 检查搜索关键词拼写，尝试用通配符或 OR 放宽条件。
- 勾选“忽略大小写”或检查是否关闭了此选项。

### 2. PDF 页码定位不准？
程序使用 `pdf-extract` 提取文本，某些 PDF 可能分页不准确。可以尝试更换 PDF 提取库（需修改源码 `extract_pages` 函数）。一般官方发布版会尽量适配常见 PDF。

### 3. 文件索引很慢？
- 大文件或大量文件需要一定时间，可在索引过程中随时暂停。
- 可检查日志中是否有错误文件记录（`error.log`）。

### 4. 如何备份数据库？
复制整个 `~/.doc_searcher/db/` 目录即可。

### 5. 如何更换目标文件夹？
目前需要新建一个数据库并指定新目录，原数据库可保留或删除。
